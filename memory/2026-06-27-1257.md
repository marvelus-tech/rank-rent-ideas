## Root cause of the repeated failures

**Syntax error in `renderTeardowns()`** — a missing `});` to close the `filtered.forEach()` loop. This caused the entire JavaScript file to fail parsing, so **no data rendered at all** — not just teardowns, but lots, builders, everything.

---

## How to prevent this in the future

**1. Always validate syntax before pushing**
```bash
# Extract JS from HTML and check it
sed -n '/<script>/,/<\/script>/p' index.html | sed 's/<script>//' | sed 's/<\/script>//' > /tmp/check.js
node --check /tmp/check.js
```

**2. Test the live URL after every push**
GitHub Pages caches aggressively. Use `?nocache=1` to bypass.

**3. Add a CI check** (GitHub Actions) to validate HTML/JS on every commit:
```yaml
# .github/workflows/validate.yml
- name: Validate JavaScript syntax
  run: |
    sed -n '/<script>/,/<\/script>/p' index.html > /tmp/script.js
    node --check /tmp/script.js
```

**4. Use a build step** with a bundler (Vite, Parcel) that catches syntax errors at build time instead of at runtime.
