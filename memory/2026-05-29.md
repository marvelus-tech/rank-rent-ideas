## HyperFrames Sizzle Video — Marvelus.cc

Built a 40-second marketing sizzle video for Marvelus.cc using HyperFrames (HTML/GSAP → MP4).

### What was built
- 5-scene composition: Search Typing → Browser Mockup → How It Works → Stats → CTA
- 1920×1080, 30fps, dark navy/cyan aesthetic
- File: `~/.openclaw/workspace/hyperframes-sizzle/marvelus-sizzle/marvelus-sizzle-v4.mp4`

### Bugs fixed along the way
1. **Scene opacity bug**: Scenes 2-5 had `opacity: 0` in CSS which overrode HyperFrames clip visibility — content was "present" but invisible. Fixed by removing scene-level opacity and animating inner elements only.
2. **Stray comma bug**: Previous edit left hanging commas from removed `tl.set()` calls, breaking the entire GSAP timeline. Had to rewrite the script section cleanly.
3. **Google Fonts don't load** in headless Chromium render — falls back to system sans. Non-blocking but noted.

### Key learnings
- HyperFrames uses `data-start`/`data-duration` attributes for clip timing; CSS `opacity` on the clip itself conflicts with the engine's visibility management.
- When editing GSAP timelines, removing `.set()` calls requires removing their trailing commas too — JS is unforgiving about trailing commas inside argument lists.
- Font loading in headless renders is unreliable; local font files or `@font-face` with base64 embedding is the robust path.
