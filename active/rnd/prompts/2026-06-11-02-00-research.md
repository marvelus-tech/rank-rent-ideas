## Critical Lessons Learned

### GitHub Pages Caching is Aggressive
- GitHub Pages caches content for 5-10 minutes even after push
- Raw GitHub content (raw.githubusercontent.com) updates immediately
- But github.io pages lag behind
- Solution: Add cache-busting query params (?nocache=N) or force empty commits
- Even then, may need to wait 5-10 minutes for propagation

### Git Repository Structure Matters
- The workspace is a monorepo with multiple dashboard subdirectories
- Each subdirectory pushes to a different GitHub repo (marvelus-tech/REPO-NAME)
- GitHub Pages serves from repo ROOT, not subdirectories
- File must be at `index.html` (root), not `subdir/index.html`
- Had to move files from subdirectories to root for each repo
